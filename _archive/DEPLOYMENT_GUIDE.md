## рџљЂ End-to-End Workflow (5 Steps)

### **Step 1: Upload Fonts (Webapp)**

```
http://localhost:5001 в†’ Explore Tab в†’ Upload Font
```

**Actions:**
1. Click "Upload Font" button
2. Select TTF/OTF file from disk
3. Font is processed and saved to `training_data/{FontName}/`
4. Metadata (unicode, glyphs) stored in `training_data/{FontName}/meta.json`

**Check Result:**
```bash
ls -la training_data/
# Should show: Roboto/, Noto_Sans_Devanagari/, etc.
```

---

### **Step 2: Generate Training Manifest**

```bash
python3 unified_pipeline_manager.py --prepare

# Output:
# вњ… Manifest created: training_data/training_manifest.json
#    Fonts: 2
#    For Colab: train_unified_pipeline.ipynb
```

**What it does:**
- Scans `training_data/` for all fonts
- Creates JSON manifest listing fonts, unicode ranges, script types
- Manifest is used by Colab for training

**Check Result:**
```bash
cat training_data/training_manifest.json | jq .
# Shows: fonts, total_fonts, scripts
```

---

### **Step 3: Train in Google Colab** 

**Upload to Colab:**
1. Get `train_unified_pipeline.ipynb` from project root
2. Upload to Google Colab
3. Upload `training_data/training_manifest.json` to Colab environment
4. Run all cells

**Key Parameters (adjust as needed):**
```python
LATENT_DIM = 64              # Latent space dimensionality
EPOCHS = 50                  # Training iterations
BATCH_SIZE = 8               # Batch size
LEARNING_RATE = 0.001        # Adam LR
KL_ANNEAL = True             # Enable KL annealing (recommended)
```

**Expected Output:**
- Checkpoint file: `font_vae_unified.pt`
- File size: ~50-80 MB
- Metrics: loss, kl_loss, recon_loss curves

**Download Model:**
```
Download в†’ font_vae_unified.pt (or renamed to font_vae_v2.pt)
```

## рџ”Ќ Troubleshooting

### **Problem: Models dropdown is empty**

**Check:**
```bash
cat models/model_registry.json
# Should show models array with entries
```

**Fix:**
```bash
# Regenerate registry
python3 unified_pipeline_manager.py --status
# If still empty, manually create:
echo '{"latest": null, "models": {}, "legacy": []}' > models/model_registry.json
```

### **Problem: Font dropdown is empty**

**Check:**
```bash
ls training_data/
# Should show font directories like Roboto/, etc.
```

**Fix:**
- Use webapp Explore tab to upload fonts
- Or manually create directories with meta.json

### **Problem: Generation fails with "Model not found"**

**Check:**
```bash
ls -la models/font_vae_unified.pt
# Should exist and point to versions/vX/model.pt
```

**Fix:**
```bash
python3 unified_pipeline_manager.py --register-model /path/to/model.pt v1
```

### **Problem: Generated glyphs all look identical**

**Cause**: Old model without per-character encoding

**Fix**: Train with latest `train_unified_pipeline.ipynb`

### **Problem: Server crashes with "CUDA out of memory"**

**Check**: Running inference on CPU-only machine with large model

**Fix**:
```python
# In inference code, use CPU explicitly
model.eval()
with torch.no_grad():
    # Use device = 'cpu'
```
